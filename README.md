# TeleOps Assistant 电报频道/群组运营助手

TeleOps Assistant 是一款面向 Telegram 内容运营场景的 Windows 桌面工具，适合用于自有频道、群组或已获授权素材的整理、搬运、发布和广告投放。软件重点解决多账号、多频道、多任务并行运营时的效率、稳定性和可追踪性问题。

## 主要功能

### 账号与代理管理
<img width="1280" height="809" alt="photo_2026-06-05_12-20-09" src="https://github.com/user-attachments/assets/39939a98-5e37-47be-ba92-4b0df834bb6d" />
- 统一管理多个 Telegram 账号，支持账号名称、手机号、验证码、两步验证码和代理配置。
- 支持为账号配置独立代理，也可在未填写账号代理时使用系统代理。
- 支持账号测试，检查代理连接、账号登录状态、账号健康状态和基础可用性。
- 账号列表支持编辑、测试、删除，正常使用时默认隐藏输入框，减少误操作。

### 频道与群组任务
<img width="1280" height="809" alt="photo_2026-06-05_12-20-12" src="https://github.com/user-attachments/assets/4b5eb241-3161-48ed-ac4a-2038e494bc86" />
- 支持配置来源频道/群组和目标频道/群组，兼容 `@用户名`、`t.me` 链接和 `-100` 开头的频道 ID。
- 支持一个任务绑定多个账号，多个任务也可以共用同一账号，方便按任务灵活分配账号资源。
- 任务列表支持启动、暂停、编辑、复制、删除、锁定、批量启动、批量停止和批量重试。
- 支持拖动排序任务，便于把常用任务放在前面管理。

### 资源下载
<img width="1280" height="809" alt="photo_2026-06-05_12-20-14" src="https://github.com/user-attachments/assets/59d74101-c460-46d9-a3c5-d45af14e15fc" />
- 支持从来源频道或群组下载图片、视频和文件到本地指定文件夹。
- 支持随机选取、顺序选取、按日期到最新三种资源选取方式。
- 支持设置开始处理的帖子 ID，清空资源队列后可按用户设置重新开始。
- 支持按帖子整组处理，相册或媒体组按一条帖子计算。
- 支持最多 5 个资源并行下载，提升批量资源下载效率。
- 下载文件按来源频道和帖子 ID 命名，减少多频道下载时的文件名冲突。

### 资源搬运
<img width="1280" height="809" alt="photo_2026-06-05_12-20-17" src="https://github.com/user-attachments/assets/0dbbf6cb-7e68-4af0-a4a3-45fabcccf350" />
- 支持直接转发模式：由 Telegram 服务器转发媒体，节省代理流量，不修改媒体和配文。
- 支持重新上传模式：下载媒体后重新上传，可应用内容规则和媒体处理设置。
- 支持按高级筛选条件选取资源，只搬运符合条件的帖子。
- 支持实时监听模式，持续监听来源频道或群组的新消息，并自动处理后续新增内容。
- 普通调度模式只处理本轮任务，不会持续同步新增帖子，避免非监听任务超出预期运行。

### 处理发布
<img width="703" height="1280" alt="photo_2026-06-05_12-20-19" src="https://github.com/user-attachments/assets/95993a37-211c-4213-8cc2-1f8d0dba7561" />
- 支持图片和视频在发布前进行二次处理，例如加水印、整图模糊、高清修复、品牌化重包装等。
- 支持图片水印和视频水印，包含文字水印、图片水印、频道角标、统一边框、底部来源条和日期标签等样式。
- 支持视频剪辑，可设置片段数量和片段时长；片段数量为 0 时保留完整视频。
- 支持视频动态水印位置，例如固定四角、顶部滚动、中部滚动、底部滚动和全屏飘动。
- 支持处理后按整组发布，避免一条帖子中的多个媒体被拆成多条消息。

### 高级筛选

- 支持按资源类型筛选文字、图片、视频和文件。
- 支持按资源大小、视频时长进行筛选，整组帖子中只要有资源不符合条件即可丢弃整帖。
- 支持包含关键词和排除关键词，用 `|` 分隔多个关键词。
- 支持忽略带按钮的帖子、忽略无文本的帖子，减少不适合搬运的内容进入队列。
- 支持实时监听任务和普通任务分别使用对应的筛选逻辑。

### 内容规则

- 支持批量替换规则，每行一条，格式为 `替换前|替换后`，替换后留空表示删除原文。
- 支持尾巴 HTML，可快速插入粗体、斜体、下划线、删除线、代码块、链接、引用、换行等标签。
- 支持资源防剧透、照原文搬运、清除文本中的链接和丢弃原文。
- 支持清除正文中的普通链接、HTML 超链接、Telegram 链接和 `@用户名` 链接。
- 直接转发模式下自动隐藏不会生效的内容规则设置，避免配置误解。

### 评论区搬运

- 支持搬运帖子评论区中的文字、图片、视频和文件资源。
- 支持设置帖子搬运上限，控制每条主帖处理的评论数量。
- 支持沿用高级设置，也支持为评论单独配置自定义筛选规则。
- 支持为评论单独配置内容规则，包括替换规则、尾巴 HTML、丢弃原文等。
- 评论资源会进入独立评论队列，支持清空完成、重试失败和失败排查。

### 广告投放
<img width="1280" height="809" alt="photo_2026-06-05_12-20-24" src="https://github.com/user-attachments/assets/1113995f-14c4-4afe-9427-2c67aa9847cb" />
- 支持创建多个广告投放配置，独立管理广告名称、投放频道、发送账号、频率和有效期。
- 支持从频道任务中提取目标频道，也支持手动输入多个频道或群组。
- 支持广告正文 HTML，支持本地图片、视频和文件资源。
- 支持间隔执行、每天定时、指定时间段投放。
- 支持立即试发广告，并在试发期间防止重复点击导致重复发送。
- 支持广告置顶、删除历史广告和尾随频道消息发布。

### 后台运行与任务恢复

- 支持关闭窗口后保留系统托盘并继续运行后台任务。
- 支持实时监听任务持续运行，直到用户手动暂停。
- 网络或授权服务临时异常时，任务会暂停提交发布，等待恢复后继续处理未完成队列。
- 用户手动暂停的任务不会被自动恢复，避免恢复逻辑干扰人工操作。
- 任务运行、资源队列、评论队列和广告投放都会记录状态，便于排查问题。

### 仪表盘与日志
<img width="1280" height="809" alt="photo_2026-06-05_12-20-06" src="https://github.com/user-attachments/assets/1c5efc70-b90c-4191-85be-ae14ba1877af" />
- 仪表盘展示今日发布、代理流量、活跃任务、异常任务、任务趋势等运营数据。
- 支持按一天、一周、一月、一季、一年查看统计。
- 日志告警记录任务启动、完成、失败、代理异常、Telegram 异常、资源下载/上传异常等关键事件。
- 日志最多保留三天，自动清理过期记录，避免本地数据长期膨胀。
